LimitRePathTo: Limit Requests Based on File Extension

Limit serving of files to the specified set of regular expressions. If the file is not one of the specified paths, then reject the request.

Configuration

You can provide a simple list of extensions that when matched will be served. All other paths return a HTTP Not Found (404) error.

{
    "servername": { 
        "listen_to":[ "http://www.example.com" ],
        "plugins":[
        ...
            { "LimitRePathTo": { 
                "Paths": [ "^/.*\\.html" ]
            } },
        ...
}

Full Example

{
     "www.zepher.com": { "LineNo":2,
        "listen_to":[ "http://www.zepher.com:3210/" ],
        "plugins":[
            { "LimitRePathTo": { 
                "Paths": [ "^/[a-z][a-z]/", ".html$" ]
            } },
            { "file_server": { "LineNo":9,
                "Root":"./www.zepher.com__3210",
                "Paths":"/"
            } }
        ]
    }
}
