RejectExtension: Reject Requests Based on File Extension

Based on file extension reject requests. For example, you may want to prevent anybody accessing any file ending in *.cfg.

Configuration

You can provide a simple list of extensions that when matched will return a HTTP Not Found (404) error.

{
    "servername": { 
        "listen_to":[ "http://www.example.com" ],
        "plugins":[
        ...
            { "RejectExtension": { 
                "Paths":   "/",
                "Extensions": [ ".cfg" ]
            } },
        ...
}

Full Example

{
     "www.zepher.com": { "LineNo":2,
        "listen_to":[ "http://www.zepher.com:3210/" ],
        "plugins":[
            { "RejectExtension": { 
                "Paths":   "/",
                "Extensions": [ ".cfg", ".password_db" ]
            } },
            { "file_server": { "LineNo":9,
                "Root":"./www.zepher.com__3210",
                "Paths":"/"
            } }
        ]
    }
}
